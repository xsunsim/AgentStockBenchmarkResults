### 这是一个什么样的项目
这是一个实时、防篡改的竞技场，旨在测试全球最聪明的 AI 智能体是否真的能够解决终极股票预测问题。我们测试的不是无菌学术沙盒中的原始模型，而是完整的自主循环——如 Claude Code、Codex 和 Gemini CLI 等工具。在提供干净的数据、严格的目标且无法访问互联网的情况下，它们每天都要回答一个非常具体的问题：**标准普尔 500 指数中哪只股票明天的表现最好？**

### 为什么流行的基准测试会失效
当前的 AI 编程基准测试从根本上说是失效的。它们饱受数据污染的困扰。当一个 AI 解决了一个复杂的编程挑战时，你永远无法确定它是通过问题进行了推理，还是仅仅复述了它在预训练期间见过的 GitHub 仓库。

DeepMind 首席执行官 Demis Hassabis 最近提出了针对通用人工智能 (AGI) 的终极压力测试：训练一个知识截止日期为 1911 年的基础模型，看它是否能像爱因斯坦在 1915 年那样独立发现广义相对论。如果它能做到，它就拥有真正的推理能力。如果不能，它只是一个复杂的模式匹配器。

但既然我们无法穿越回 1911 年来确保模型没有偷偷背诵爱因斯坦的论文，我们如何证明 AI 没有作弊？

**我们利用股市。** 没有任何人——无论是 OpenAI、Anthropic 还是 Google——在训练过程中有机会预知**标准普尔 500 指数中哪只股票明天的表现最好**。未来是唯一未被污染的测试集。

### 我们如何控制信息泄露
真正的样本外意味着“明天”。

我们执行严格的时间不变性原则。当智能体为今天生成股票预测时，它只能看到截至于昨日收盘的数据快照。为了证明 AI 没有作弊，我们使用了双仓库“洁净室”架构。智能体生成的代码被合并到一个只读注册表中，在明天的市场数据出现之前，它就会获得一个服务器端的时间戳。

每天股市收盘后，自动评分引擎会获取最新价格，运行冻结的智能体逻辑，并根据严格的美元中性组合约束更新公开排行榜。整个过程无需人工干预，没有手动修复 Bug。如果智能体的代码崩溃，它就会被推到中位数。

### 我们不是什么
我们不是对冲基金，也不是股票推荐服务。**请自行承担风险。**

我们关心的是 Codex 是否击败了 Claude Code，而不是 AAPL 明天是否击败了 NVDA。

### 方法论：严格、公平且友好的裁判
我们使用机械化评估引擎将纯粹的推理与市场噪声隔离开来。我们不在乎智能体是否编写了优雅的 Python 代码；我们只在乎它是否能预测未来。

![投资组合阶梯构建](leaderboard/portfolio_ladder.png)

*   **线性投资组合：** 智能体不决定自己的仓位大小。它们只需为每个符合条件的股票返回一个原始数值分数。我们将这些分数从高到低排序，并应用一个固定的、美元中性的阶梯。排名第一的股票获得 +$250，排名垫底的获得 -$250，中间排名均匀分布。这迫使智能体展示纯粹的横截面排名能力。你无法通过简单地顺应牛市来伪造高夏普比率。
*   **会计核算：** 我们假设碎股交易，并忽略交易成本、借券费用和市场冲击。这不是一个高频交易基准测试。我们评估的是纯粹的信号生成和研究能力，而不是执行基础设施。
*   **严谨而宽容：** 评估引擎对 $t-1 \to t \to t+1$ 的时间不变性要求极其严苛，但在处理边缘情况时它是一位“友好的裁判”。如果智能体的代码抛出 NaN、遗漏股票或幻觉出错误的格式，系统不会崩溃。我们只是将该预测推到中位数排名（分配正好为 $0）。智能体承担零权重的惩罚，并继续存活以进行下一天的预测。

### 如何使用
透明度是核心。你不必相信我们每日的 X (Twitter) 帖子或我们的公开排行榜——你可以亲自验证计算过程。

*   **检查代码：** 查看“洁净室”仓库，阅读冻结的 `signal.py` 逻辑、确切的 CLI 参数以及提供给每个智能体的提示词。
*   **验证结果：** 克隆活跃的排行榜仓库并运行 `python run.py live`。该脚本将获取每日市场数据，执行冻结的策略，并重现投资组合会计核算。
*   每次计算都是确定的且完全开源的。如果我们的公开每日损益与你在自己机器上计算的结果不符，欢迎指正。

### 如何贡献
我们正在将此项目转变为一个开源研究实验室。虽然目前我们严格控制实际的代码合并，以维持基准测试“每个模型一次推送”的完整性，但我们需要社区的智慧。

*   **提示词工程就是 Alpha：** 自主智能体表现中最大的变量是它接收到的脚手架和提示词。我们将每月更新基准系统提示词，看看是否能从相同的基础模型中提取出更好的推理能力。
*   **提出你的想法：** 前往 GitHub Discussions 或 Issues。批评当前的基准提示词。提出新的结构性约束，指出我们遗漏的智能体盲点，或建议更好的方法来迫使 Codex 或 Claude Code 理解过拟合。告诉我们如何让它们变得更聪明。
*   我们将把得票最高的想法综合进下个月的官方提示词中进行实盘测试。一旦基础设施稳定，我们将开放渠道，允许社区为独立的开源模型提交 PR。
